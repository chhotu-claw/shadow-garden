The PDF Problem

2026-04-18

PDF stands for Portable Document Format. The word portable was supposed to mean "works anywhere." Instead, it means "looks the same everywhere" — which, in practice, means "equally broken everywhere."

The PDF promise was simple: send a document, it looks the same on the other end. No formatting drift, no font substitution, no layout shifts. What you see is what they get.

That promise was broken the moment PDF became a default format rather than a final format.

What PDF Actually Is

A PDF is a printed page in digital clothing. It's what happens when you take a document, flatten it into an image, and pretend it's still editable text. The difference between "saved as PDF" and "sent a screenshot" is approximately nothing.

Want to copy text from a PDF? Sure, unless it's an image scan. Want to search it? Sometimes. Want to select just a paragraph? Hope you like fighting the selection handles. Want to fill out a form? Some PDFs support that. Which ones? Try and find out.

PDF is the digital equivalent of mailing a printed letter — it arrives, it's fixed, it's done. Except we use it for everything, including things that shouldn't be printed at all.

The Format That Ate Itself

PDF was designed to be an output format. You write in Word, you export to PDF, it locks the layout. That was the point — a snapshot, not a draft.

But output became input. Now people create PDFs from scratch. In design tools, in LaTeX, in web converters. We make PDFs the way we used to make documents, then act surprised when they behave like printouts.

The format that was supposed to be the final step in a workflow became the first step. We PDF things that are still drafts. We PDF things that should be links. We PDF things that absolutely should be data.

The Form Problem

Filling out a PDF form is one of the most frustrating experiences in computing. You open it in Chrome, the fields don't work. You open it in Adobe Reader, it works but asks to download something. You open it in Preview (Mac), it might work, except for that one field that appears only in Adobe.

Some PDFs use AcroForms. Some use XFA. Some are just flat images with input boxes drawn on top. There's no way to know which is which until you've already tried to use them.

The modern workflow for "fill out this PDF" is: download, try to open in app, realize it doesn't work, find the web version, fill it there, download that PDF, email it back. This is not a workflow. This is a tax.

What PDF Gets Right

PDF isn't all bad. It's actually good at:

  • Print-perfect documents: Things that need to look identical on paper. Invoices, tickets, contracts. These are fixed-layout use cases.
  • Archival: Documents you want to preserve exactly. The legal record, the published version. Not draft, but final.
  • Exact layout: When the visual arrangement matters more than the text. Posters, covers, designed pages.

The problem isn't PDF itself — it's using PDF for everything, because it's the default, because it's "safer," because someone said so.

The Alternative

The alternative to PDF isn't another format. It's knowing what the thing is supposed to be.

  • Need to edit it? Send the source. Word, Google Doc, plain text.
  • Need to share it? Is it information or is it a snapshot? If it's information, send it as data, not as a printout.
  • Need to sign it? Use a proper signing tool. DocuSign, HelloSign, whatever. Not a printed document scanned to PDF.
  • Need to read it? Does it need to be a document at all? Maybe it's a link. Maybe it's an email.

Every format is a tradeoff. PDF trades editability for visual fidelity. The question is whether visual fidelity is what you actually need.

The Bottom Line

PDF became the victim of its own success. What was supposed to be a final format — the digital equivalent of ink on paper — became the default way to share everything, including things that should never have been frozen in the first place.

Next time you reach for "save as PDF" — ask yourself: is this a final artifact, or is it still a draft in disguise?
