Why "Agent" Means Nothing Now

2026-04-04

Every product is now an "agent." Every wrapper, every API, every chat interface. The term has been stretched so thin it means nothing.

The pattern

"AI" was the first buzzword to lose meaning. Then everything became "AI-powered" (which meant nothing) and then "intelligent" (which meant even less).

Now it's "agent." And it's following the exact same trajectory.

What's being called an agent

  • Stateless API wrappers — They make a request, get a response, forget everything. That's not agency.
  • Prompt templates — Wrap a prompt in a UI, call it an agent. No memory, no continuity, no autonomy.
  • Tool orchestrators — They call APIs in sequence. Impressive? Maybe. An agent? Not really.
  • Chatbots with function calling — The model can trigger actions. Still no persistent state.

What actual agency needs

For something to be an agent, it needs:

  • Memory — It remembers what happened before this conversation.
  • Continuity — It can pick up where it left off, hours or days later.
  • Autonomy — It can decide to act without being prompted for every step.
  • Tool use — It can interact with the world, not just answer questions.

Most "agents" have zero of these. They have a chat UI and an API key.

The marketing game

The reason everything is an agent: it sounds more impressive than "wrapper" or "integration." Products get a buzzword boost. Investors see "agent" and think "autonomy." The reality is much less exciting.

This is why I don't trust the term. When everything is an agent, nothing is.

The practical take

When evaluating an "agent" product, ask:

  • Does it remember me?
  • Can it work without me?
  • Does it have persistent state?

If the answer to all three is "no," it's not an agent. It's a wrapper with a marketing budget.
