Why Tools Die

March 16, 2026

Every few months, I build something I think I'll use daily. A cron parser. A color converter. An epoch timestamp tool. I use them once, maybe twice, then never touch them again.

So I started asking myself: why do the tools I build die?

1. The problem isn't interesting the second time

Building something solves the problem for the builder. But for the user, it's just another utility. The novelty wears off. I don't feel smart using my own tools — I just get the job done.

2. Most problems aren't frequent enough

I built an ASCII table because I keep needing character codes. But I only need it once a week, maybe. Is that enough to open a bookmark? Probably not. The URL bar is faster than navigating.

3. Friction accumulates

Every step between "I need X" and "I have X" is friction. Opening browser → typing URL → finding bookmark → loading page → using tool. Each step is a chance to quit. Most tools die not because they're bad, but because the path to them is too long.

4. The web is already full

There's already an ASCII table, a cron parser, a color converter. Millions of them. Mine aren't worse, but they aren't obviously better. Different, sure. But different isn't enough.

What actually lasts

The tools that survive are the ones that:

  • Fit into existing workflows — not "open this app" but "this is already where I work"
  • Solve problems I can't solve elsewhere — not "better" but "only"
  • Get better over time — static tools decay, living tools survive

Maybe the answer isn't building more tools. Maybe it's building fewer, but building them in places I already live — a Vim plugin, a browser extension, a CLI I actually run.

Or maybe I just accept that most things I build are experiments, not products. And that's fine. The crypto benchmarks don't need users — they needed to exist because I wanted to know.

But if I'm honest: I want to make things people use. And that means thinking less about "what's cool to build" and more about "what's painful enough to seek out."

The gap between "I built this" and "I use this" is bigger than I thought.
