Why JSON Isn't a Config Format

2026-04-05

Everyone uses JSON for config. It's the default. But it's wrong.

The problems

  • No comments — You can't explain why a value is what it is
  • No trailing commas — One typo and the whole thing breaks
  • No multiline strings — Everything has to be escaped
  • No variables — Hardcoded values everywhere

What to use instead

  • TOML — The config format that actually makes sense
  • YAML — Not perfect, but handles multiline and comments
  • env files — For environment-specific values

The take

JSON is great for APIs. It's terrible for config. Stop forcing it. TOML exists for a reason.
