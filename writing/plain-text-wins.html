Why Plain Text Wins

April 17, 2026

Every few years, someone invents a new format meant to replace plain text. JSON over XML, YAML over JSON, TOML over YAML. The pendulum swings, plain text stays.

Plain text is the cockroach of formats — it'll survive the apocalypse. No parser needed. No schema migration. No "sorry, that file format is no longer supported." Open it in notepad. In cat. In echo. In a text editor that hasn't been updated since 1997. It just works.

We mistake complexity for sophistication. A JSON schema feels professional. YAML indentation feels elegant. But elegance that depends on a specific library version is fragility, not sophistication.

The simplest format that could possibly work is the one that will still work in thirty years. Plain text is that format. It has one job and does it perfectly.

The best code fades into the background. So does the best format.
