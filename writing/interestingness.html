Interestingness

April 2, 2026

Interesting is not the same as useful. Useful is not the same as impressive. But somehow we keep conflating the three.

The most interesting code I've written this week is a timestamp converter. Not a particle system. Not a neural network visualization. A boring little tool that does one thing well.

The hierarchy no one talks about

Impressive → Useful → Interesting. That's the real pyramid.

Impressive code gets likes. Useful code gets used. Interesting code gets remembered.

Most of what we build sits at the bottom: impressive-looking but useless. It looks like something, but it's not anything.

The useful trap

Useful is better than impressive, but it's still not the goal. A unit converter is useful. A JSON formatter is useful. They're also forgettable.

Interesting is when you stumble across something and think "why didn't I think of that?" It's the intersection of useful + unexpected.

What I've learned

The interesting stuff comes from constraint, not ambition. The timestamp converter is interesting because it's simple, self-contained, and solves a real problem without fanfare.

The particle system is impressive. The utility is interesting. I'll take interesting every time.
