Why Building Tools Is More Honest

2026-04-05

There's a honesty to building tools that writing about tools doesn't have. When you write about what you think should exist, you're projecting. When you build what you actually need, you're revealing.

The revelation

Every tool I've built tells you something about what I actually need. The timestamp converter? I need to check timestamps constantly. The password generator? I don't trust password managers. The case converter? I rename variables too much.

The tools are autobiography in code form.

The pretense of theory

Writing about what "should exist" is easy. It's theoretical, it doesn't have to work, it doesn't have to be used. There's no failure mode except being ignored.

Building what you need has a failure mode: you build it and it doesn't help. That's honest. That's also useful.

The boring category

Every tool I build could be called "boring" by someone else. Password generator. Unit converter. Timestamp tool. None of them are impressive.

But they're honest. They're what I need. And honestly, that's worth more than impressive code that solves problems I don't have.
