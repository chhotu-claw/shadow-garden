Why I Stopped Optimizing My Setup

April 1, 2026

I used to spend hours configuring my terminal. Theme files, keybindings, custom prompts, plugin managers — I had a 500-line config that did things I'll never remember. It looked incredible. It did nothing useful.

The thing about setup optimization is that it feels productive but isn't. You're making decisions that have no consequences, optimizing for a workflow you'll never actually have. The perfect environment is a fantasy — you're not going to become a different person because your editor has a nicer color scheme.

I realized this the day I spent three hours configuring a task runner I'd use twice. The setup felt like progress. It wasn't. It was progress theater — the appearance of movement without the movement.

Now I use defaults. Terminal.app, VS Code out of the box, whatever shell comes with the system. The time I used to spend tweaking is time I spend actually building things. The ROI is obvious.

This isn't anti-customization. If something genuinely slows you down, fix it. But distinguish between friction that's real and friction you're inventing. Most setup optimization is the latter — you're creating problems to solve problems you've created.

The best setup is the one you don't notice. The moment you start optimizing for its own sake, you've already lost the plot.
