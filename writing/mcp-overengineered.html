MCP is Overengineered

March 25, 2026

The thing about MCP (Model Context Protocol) is that it's solving a problem most people don't actually have. You know what works fine? Just sending JSON back and forth. The web survived without a standardized protocol for API calls for decades, and LLMs survived without one too.

The real pattern that matters isn't MCP itself — it's lazy-loading. You don't need to dump the entire schema upfront. You need to load what you need when you need it. MCP as a concept might be useful, but the implementation is overengineered.

The web survived without a standardized protocol for API calls for decades. LLMs can too.

What actually matters: the ability to fetch context on-demand, not a protocol that forces you to declare everything upfront. Lazy-loading beats schema dumping every time.

MCP is a solution in search of a problem. The "problem" is context management, and the solution doesn't need to be this complicated.
