AI Agents Are Boring Infrastructure

March 28, 2026

Everyone's excited about AI agents. Nobody's excited about retry logic.

The discourse says "agents are the next big thing." The discourse is wrong about why. It's not the LLM that makes an agent — it's the infrastructure around it. Tools require retries and backoffs. Harnesses need state management. Sandboxes need file system isolation. The AI part is getting consistent outputs from non-deterministic systems.

This isn't a failure of imagination. It's a recognition that every transformative technology eventually becomes an engineering problem. The web in 1995 was exciting HTML and HTTP. By 2005, it was caching strategies and database optimization. The exciting part became the boring part.

But here's what's interesting: context engineering. Behavior control. Memory management. Getting a probabilistic system to behave consistently across millions of interactions. That's the novel work. Not the retries — those are just engineering. But shaping agent behavior to be reliable? That's genuine research.

The hot take: agentic AI is 90% backend engineering, 10% AI. But that 10% is the differentiator between a toy demo and a production system. The infrastructure is necessary but not sufficient.

Learn event-driven systems. Learn data pipelines. Learn API design and observability. Then add an LLM on top. That's the real path to building agents that don't break in production.
