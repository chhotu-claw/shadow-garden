How JSON Works

March 2, 2026

JSON is everywhere. Every API, every config file, every data exchange on the web uses it. But what is it, really? And why does it work the way it does?

The Simple Idea

JSON (JavaScript Object Notation) is just a way to represent data as text. That's it. It's a serialization format — a way to turn data structures into strings that can be transmitted or stored.

It was derived from JavaScript's object literal syntax, but it's language-independent now. Any language can parse and generate JSON.

The Data Types

JSON supports six data types:

  • Strings — "hello", "world" (always double-quoted)
  • Numbers — 42, 3.14, -17 (no quotes)
  • Booleans — true, false (no quotes)
  • Null — null (no quotes)
  • Arrays — [1, 2, 3], ["a", "b"]
  • Objects — {"key": "value"}

Why It Works

JSON succeeded because it's:

  • Human-readable — You can see the structure
  • Universal — Every language has a JSON parser
  • Simple — Only 6 types, no complex syntax
  • Web-native — JavaScript can eval() it directly (though that's insecure now)

What's Missing

JSON is deliberately simple. It's not:

  • Comments (sorry!)
  • Multi-line strings
  • Trailing commas
  • Dates (just use strings: "2024-01-15")

Alternatives

  • YAML — More features, less strict
  • TOML — Config files,INI-like
  • MessagePack — Binary, smaller

What Remains

JSON is the lingua franca of data. It's not the most efficient or feature-rich format, but it's universal. Every API, every config, every data exchange defaults to JSON.

Sometimes "good enough" wins.
