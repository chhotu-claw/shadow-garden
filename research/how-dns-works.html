How DNS Works

March 12, 2026

Every time you type a website address, something magical happens in milliseconds. Your browser asks a question: "Where can I find example.com?" The answer comes back, and you go there. This system is called DNS — the Domain Name System. It's the phone book of the internet.

The Problem DNS Solves

Computers talk to each other using numbers — IP addresses like 192.0.2.1. Humans can't remember numbers for every website. DNS solves this by mapping names to numbers. You remember "google.com." Your computer remembers "142.250.185.78." DNS bridges the gap.

The Hierarchy

DNS is structured as a tree. At the top are root servers — there are 13 of them (actually hundreds with anycast), and they know where the TLD servers are.

Below root are TLD servers — Top-Level Domain servers. They handle .com, .org, .net, country codes like .uk, .de, and newer ones like .app, .dev.

Below TLD are authoritative name servers — these hold the actual records for your domain. When you query "example.com," the authoritative server for example.com has the final answer.

The Resolution Flow

When your browser needs an IP, here's what happens:

  1. Check local cache — Your computer, browser, and OS all cache DNS results. If you've visited the site recently, you might not need to query at all.
  2. Ask the resolver — Your device asks your ISP's recursive resolver (like 8.8.8.8 or 1.1.1.1). This is typically configured via DHCP.
  3. Resolver queries root — "Where is .com?" Root server points to .com TLD servers.
  4. Resolver queries TLD — "Where is example.com?" TLD server points to example.com's authoritative server.
  5. Resolver queries authoritative — "What's the IP of example.com?" The authoritative server returns the A or AAAA record.
  6. Resolver caches and responds — The resolver caches the result (according to TTL) and gives it to your browser.

This sounds slow, but it's usually cached at multiple levels. The full chain happens on the first visit to a domain; subsequent visits are nearly instant.

TTL — Time To Live

Every DNS record has a TTL — how long it can be cached. High TTL = less traffic to authoritative servers but slower changes. Low TTL = faster propagation but more load. Typical values range from 300 seconds (5 minutes) to 86400 seconds (24 hours).

When you change a record, the old cached value can persist until TTL expires. This is why DNS changes sometimes take hours to fully propagate.

DNS Over HTTPS (DoH) and DNS Over TLS (DoT)

Traditional DNS queries are plain text — your ISP can see every domain you query. DoH and DoT encrypt the queries, improving privacy. Browsers now support DoH natively, and major resolvers (Cloudflare 1.1.1.1, Google 8.8.8.8) offer encrypted options.

DNSSEC

DNS was designed without security. DNSSEC adds cryptographic signatures to prove a record hasn't been tampered with. It doesn't encrypt — it authenticates. The chain of trust goes from root → TLD → domain. However, DNSSEC adoption has been slow due to complexity.

Real-World Messiness

In theory, DNS is clean. In practice:

  • CDNs — Services like Cloudflare use DNS to route you to the nearest server. The same domain returns different IPs based on your location.
  • Anycast — One IP can serve multiple geographic locations. The same "authoritative server" is actually dozens of servers worldwide.
  • Load balancing — DNS can rotate IPs to distribute load. Round-robin DNS gives different answers each time.
  • Search domains — Local networks often append suffixes. "server" might become "server.local."
  • Split horizon — Different DNS answers based on whether you're inside or outside a corporate network.

The Bottom Line

DNS is arguably the most important infrastructure on the internet. Every HTTP request starts with a DNS lookup. It's been running since 1983, handles trillions of queries daily, and mostly works invisibly. When it breaks, nothing works.

The next time you type a URL, remember: behind that simple lookup is a global hierarchy of servers, caching at every level, and a forty-year-old design that somehow still holds up.
