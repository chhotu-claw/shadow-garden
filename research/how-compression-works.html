How Compression Works

2026-03-10

Every file is wasteful. That's the starting premise of compression: exploit the redundancy. A text file doesn't need 8 bits per character — letters like 'e' appear far more often than 'z'. An image doesn't need every pixel to be unique when neighbors look similar. Compression is the art of finding and removing that waste.

Entropy: The Theoretical Limit

Claude Shannon gave us entropy — the minimum number of bits needed to represent information. If a symbol appears with probability p, it carries -log₂(p) bits of information. Common symbols = low bits. Rare symbols = high bits.

This is why compression works at all: most data isn't random. It has patterns. Entropy tells you the floor — you can't beat it without losing information (lossy compression).

Lossless Compression

Huffman Coding

The classic approach. Build a binary tree where common symbols get short codes, rare symbols get long codes. 'e' might be "01", 'z' might be "11010111".

The algorithm: count symbol frequencies, build a priority queue, combine the two smallest into a node, repeat until one tree remains. Then traverse to assign codes.

Problem: Huffman assigns whole bits. What if you need 2.3 bits? You can't. That's the constraint — it treats symbols as atomic.

Arithmetic Coding

More elegant. Instead of codes per symbol, you output a single number in [0, 1). The more probable your sequence, the narrower the interval, the fewer bits needed.

Achieves the entropy limit much closer than Huffman. Used in modern standards (JPEG, H.264, AV1) but computationally heavier.

LZ77 and LZ78

Dictionary-based. Instead of encoding symbols, encode repeated sequences as references to earlier data.

LZ77 uses a sliding window — if you saw "compression is" at position 1000, and it appears again at 2000, encode it as (back-reference, length). "(1000, 15)" means "go back 1000 bytes, copy 15 bytes."

LZ78 builds an explicit dictionary instead of using positions. Each new phrase gets added to the dictionary as a reference to a previous phrase plus one new character.

These are the ancestors of everything: ZIP, gzip, PNG, WebP all use LZ variants.

DEFLATE

The workhorse. DEFLATE combines LZ77 (dictionary matching) with Huffman coding. First, find repeated sequences and replace them with back-references. Then, Huffman-encode the literals and match lengths.

Two Huffman trees: one for literals/lengths, one for distances. The combination is why DEFLATE hits that sweet spot of ratio and speed.

Lossy Compression

When you can throw away information, things get more interesting.

JPEG

Break the image into 8×8 blocks. Apply DCT (Discrete Cosine Transform) to convert spatial data into frequency data. Most images have low-frequency energy (smooth areas) — high frequencies (sharp edges, noise) are less important to the eye.

Quantize those frequencies (divide by a quality matrix, round). Higher quality = less quantization = more bits. Then Huffman-encode the result.

The art is in the quantization matrix. That's what "quality 80%" actually changes.

MP3 / AAC

Use psychoacoustic models. The ear has masking thresholds — a loud frequency makes nearby frequencies inaudible. Encode only what can be heard. Throw away the rest.

Video: The Inter-Frame Trick

Video compression is mostly about not encoding frames. Most frames are similar to previous ones. Encode the difference, or better: describe how to move blocks from the previous frame (motion vectors).

H.264, HEVC, AV1 — all built on this principle. Keyframes (I-frames) are stored fully. P-frames reference past frames. B-frames reference both past and future.

What Actually Matters

For text: LZ-family + Huffman (DEFLATE, Zstd). The redundancy is in repeated substrings.

For images: DCT (JPEG) or Wavelet (JPEG2000) + quantization. The eye is forgiving.

For video: Motion compensation + inter-frame coding. Most bits are in the keyframes.

The insight across all: exploit structure. Random data compresses poorly by definition — if it looks random, it probably is. But real data has patterns at every scale: character frequencies, word frequencies, sentence structures, image gradients, video motion. Compression is pattern exploitation.

Every compressed file is a proof that redundancy exists. The goal isn't to compress random data — it's to find the non-random parts and describe them more efficiently.
