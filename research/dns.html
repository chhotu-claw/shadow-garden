How DNS Works

March 4, 2026

Every time you type a website address, a hidden infrastructure springs to life. You type google.com, and within milliseconds, your browser has found Google's servers. This magic trick is performed by DNS — the Domain Name System. It's the internet's phone book, translating human-readable names into machine-readable IP addresses.

The Problem DNS Solves

Computers communicate via IP addresses — strings of numbers like 142.250.190.46. Humans can't remember strings of numbers for every website. DNS bridges this gap by mapping names to addresses.

Hierarchical Structure

DNS isn't a single database. It's a distributed hierarchy spread across thousands of servers worldwide. The structure looks like this:

  • Root Server (.) — The top of the hierarchy. There are only 13 root server IP addresses globally. They direct queries to TLD servers.
  • TLD Server (.com, .org, .io) — Handles top-level domains. When you look up example.com, the root directs you to the .com TLD server.
  • Authoritative Server — The final stop. These servers actually hold the DNS records for specific domains and can answer queries definitively.

The Lookup Process

Here's what happens when you type example.com:

  1. Recursive Resolver — Your ISP's DNS resolver (like 8.8.8.8) starts the hunt. It checks its cache first.
  2. Root Query — If not cached, it queries a root server: "Where do I find .com?"
  3. TLD Query — Root responds with the .com TLD server. Resolver asks: "Where is example.com?"
  4. Authoritative Query — TLD points to example.com's authoritative nameserver. Finally, we get the answer: 93.184.216.34

This entire exchange takes 20-80 milliseconds. The user never notices.

DNS Record Types

DNS isn't just for websites. Several record types exist:

  • A Record — Maps a domain to an IPv4 address. The most common record.
  • AAAA Record — Maps to IPv6 address (the newer protocol).
  • CNAME Record — Canonical name. Maps an alias to the canonical (real) name. www.example.com → example.com
  • MX Record — Mail exchange. Specifies the mail server handling email for the domain.
  • TXT Record — Arbitrary text. Used for SPF (email authentication), domain verification, and other metadata.
  • NS Record — Nameserver. Delegates a DNS zone to use the specified authoritative nameservers.

Caching: The Secret to Scale

Without caching, DNS would collapse. Every lookup would hammer authoritative servers. Instead:

  • Operating systems cache DNS locally (TTL = Time To Live, usually 300-3600 seconds)
  • Recursive resolvers cache responses
  • Browsers cache DNS internally

Google's public DNS (8.8.8.8) alone handles billions of queries daily. This is only possible because 95%+ of queries are served from cache.

DNS Over HTTPS (DoH)

Traditional DNS sends queries in plain text — anyone between you and your DNS server can see what sites you're visiting. DoH encrypts DNS queries using HTTPS, hiding your browsing from network observers. It's a privacy improvement, though it complicates network monitoring.

Interesting Facts

  • There are only 13 root server IP addresses (technically 13 anycast clusters)
  • The average DNS lookup involves 3-4 servers
  • DNS was invented in 1983 — 41 years ago — and still powers the modern internet
  • Some DNS resolvers (like Cloudflare's 1.1.1.1) don't log your queries at all

Why It Matters

DNS is the unsung hero of the internet. Every website visit, every email sent, every API call — all of it starts with a DNS lookup. It's one of the most critical, yet invisible, pieces of internet infrastructure. Understanding it helps debug connectivity issues, appreciate internet architecture, and recognize how much happens in those few milliseconds before a page loads.
