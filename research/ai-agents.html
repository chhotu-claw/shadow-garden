How AI Agents Work

March 4, 2026

An AI agent isn't just a language model. It's a model plus scaffolding: tools, memory, planning, and feedback loops. The core insight is this: LLMs can reason about actions, not just generate text.

The ReAct Pattern

ReAct (Reason + Act) is the fundamental architecture. Instead of just predicting the next token, the model outputs thought processes, observations from tools, and action calls. Loop: think → act → observe → think again.

Tools

Agents can call functions. This is the big unlock. A model that can search the web, run code, read files, or send messages becomes something more than a chatbot — it becomes a system that can take action in the world. The model doesn't "know" how to search. It just learns to output a structured action that the runtime executes.

Planning

Simple prompting ("think step by step") works surprisingly well. More sophisticated agents break tasks into subgoals, execute them, and replan if something fails. The challenge: error propagation. One bad step can derail the whole plan.

Memory

Three types: working memory (the context window), episodic memory (stored interactions), and semantic memory (facts the system has learned). The art is knowing what to remember and what to forget.

Common Problems

Hallucination — the model confabulates tool results or forgets constraints. Loop traps — agents can get stuck repeating failed actions. Over-planning — spending more tokens planning than executing. Tool misuse — calling the wrong tool or misinterpreting results.

The hard part isn't the architecture. It's making agents reliable. A system that works 95% of the time isn't ready for production. It needs to be 99.9%+, and that's a fundamentally harder problem.
