How Hash Functions Work

March 5, 2026

Hash functions are the workhorses of modern cryptography. They take any input — a password, a file, a blockchain transaction — and produce a fixed-length "fingerprint" called a digest. The magic: you can't reverse it. Given a hash, you cannot figure out what the original input was.

The Properties

A good cryptographic hash function needs three things:

Pre-image resistance: Given a hash h, it's computationally infeasible to find any message m such that hash(m) = h. You can't work backwards.

Second pre-image resistance: Given a message m1, it's infeasible to find a different message m2 such that hash(m1) = hash(m2). No collisions.

Collision resistance: It's infeasible to find any two messages with the same hash. This is actually harder than second pre-image resistance — the birthday attack means you can find a collision in 2n/2 attempts, not 2n.

MD5: Dead and Buried

MD5 was the internet's favorite hash for years. Then researchers found collisions — two different files with the same MD5. Then they made fake SSL certificates. Now MD5 is cryptographic compost.

SHA-256: The Workhorse

SHA-256 (part of the SHA-2 family) outputs 256 bits. It's what Bitcoin uses for mining. The math involves bitwise operations, modular addition, and 64 rounds of mixing. It's slow by design — that slowness is what makes brute-force attacks painful.

bcrypt: Password Hashing Done Right

Here's the problem with SHA-256: it's too fast. A GPU can compute billions per second. bcrypt solves this by adding a "work factor" — you can make it as slow as you want. It also salts each hash, so identical passwords get different hashes. Rainbow tables don't work.

The Birthday Paradox

With 256 bits, you'd think collisions are impossible. They're not. The birthday paradox says in a set of just 2128 items, there's a 50% chance of a collision. That's why SHA-256's 128-bit collision resistance is the number that matters, not the 256-bit output.

What This Means

Hash functions are one-way doors. They're how you prove you know something without revealing it. How you verify a file hasn't been tampered with. How blockchains link blocks together. How passwords stay safe even when databases leak.

The math is elegant: bit shuffling that spreads input influence across the entire output. One bit change in the input flips roughly half the output bits. No pattern, no shortcut, no reverse.
