How Passkeys Work

March 21, 2026

Passwords are a 60-year-old band-aid that never actually solved the problem — they just moved it. Passkeys flip the model entirely: no secrets transmitted, the device proves ownership via cryptography.

The Core Idea

Instead of remembering a secret (password), you have a private key stored on your device. The server only has the public key. When you log in, the device signs a challenge with your private key — proving you have the key without revealing it.

WebAuthn and FIDO2

The standards stack:

  • FIDO2 — the overall framework
  • WebAuthn — the JavaScript API browsers expose
  • CTAP2 — the protocol between browser and authenticator

Key Pair Generation

When you create a passkey, your device generates an asymmetric key pair (typically ECDSA P-256 or Ed25519). The private key never leaves the device's secure enclave — TPM on Windows, Secure Enclave on Apple, Titan chip on Google.

The private key is physically impossible to extract. It's baked into hardware.

The Authentication Flow

  1. Server sends challenge — random bytes the client must sign
  2. Browser calls WebAuthn — navigator.credentials.get()
  3. User verifies — fingerprint, face, PIN, or device password
  4. Device signs challenge — using the private key
  5. Server verifies signature — using the stored public key

What Passkeys Actually Solve

  • No password reuse — each site has a unique key pair
  • No phishing — the private key is bound to the origin
  • No server breaches — public keys don't help attackers
  • No interception — private key never transmitted

The Hard Parts

  • Device loss — recovery is the unsolved problem
  • Cross-device — QR code transfers work but are clunky
  • Migration — moving from one ecosystem to another

The Verdict

Passkeys are the future of authentication. The infrastructure is ready — Apple, Google, and Microsoft all ship support. The remaining puzzle is recovery and migration. But the direction is clear: passwords are a transitional technology, and their days are numbered.
