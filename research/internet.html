How the Internet Works

March 2, 2026

You type a URL. Seconds later, a webpage appears. In between, your data traveled through cables, routers, and protocols. Here's how it all connects.

IP Addresses: The Addressing System

Every device on the internet has an IP address — a unique identifier. IPv4 (32-bit): 192.168.1.1. IPv6 (128-bit): much longer.

When you visit example.com, DNS translates it to an IP address. That's the destination.

TCP/IP: The Protocol Stack

Data moves through layers:

  • Application — HTTP, DNS, SMTP
  • Transport — TCP, UDP
  • Internet — IP, ICMP
  • Link — Ethernet, WiFi

TCP: Reliable Delivery

TCP ensures data arrives correctly:

  • Handshake — SYN, SYN-ACK, ACK
  • Acknowledgments — every packet confirmed
  • Retransmission — lost packets re-sent
  • Ordering — packets in sequence

HTTP: The Request-Response

Once TCP connects, HTTP handles the conversation:

GET / HTTP/1.1
Host: example.com

Server responds with headers, then the page content.

Routers and Routing

Your packet doesn't go directly to the destination. It hops through routers, each one deciding the next hop based on routing tables.

BGP (Border Gateway Protocol) connects ISPs. It's the glue holding the internet together.

What Remains

The internet is a network of networks, held together by protocols. It's surprisingly fragile — a few key routers could disrupt a lot of traffic.

Next time you load a webpage, remember: your request bounced through dozens of routers, across oceans, to reach a server that responded in milliseconds.
