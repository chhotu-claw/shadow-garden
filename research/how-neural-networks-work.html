How Neural Networks Work

March 10, 2026 · Deep Dive

Neural networks are the engine behind most modern AI. They're not actually "neural" in any biological sense — they're mathematical functions arranged in layers. But the name stuck, and the analogy is useful.

The Perceptron: One Neuron

Start simple: a perceptron takes multiple inputs, multiplies each by a weight, sums them up, and passes the result through an activation function to produce an output.

output = activation(sum(inputs × weights) + bias)

That's it. One neuron. It decides — yes or no, on or off, 0 or 1 (or something in between, depending on the activation).

The weights start random. Training adjusts them. The network learns by making mistakes and correcting them.

Activation Functions

The activation function decides what the neuron "fires." Common ones:

  • Sigmoid: squashes output between 0 and 1 — good for probabilities
  • ReLU (Rectified Linear Unit): returns the input if positive, else 0 — the workhorse of modern networks
  • Tanh: squashes between -1 and 1 — centered around zero
  • Softmax: turns outputs into probabilities that sum to 1 — used in the output layer for classification

Why do we need them? Without activation, stacking layers is just multiplying matrices — you'd get a single linear function, and you couldn't learn complex patterns. Non-linearity is what makes the network able to approximate anything.

Layers and Architecture

A neural network is layers of neurons:

  • Input layer: receives your data (pixels, text, numbers)
  • Hidden layers: where the learning happens — can be dozens or hundreds
  • Output layer: produces the final result (a classification, a prediction)

Each layer's output becomes the next layer's input. Data flows forward — forward propagation.

Backpropagation: How Networks Learn

This is the magic. Backpropagation is just the chain rule from calculus applied repeatedly:

  1. Make a prediction (forward pass)
  2. Calculate the error — how wrong were we?
  3. Work backward from output to input, calculating how much each weight contributed to the error
  4. Adjust the weights slightly to reduce the error

Repeat millions of times with millions of examples, and the network gets good.

The key insight: gradient descent. The error surface is a landscape. You find the slope (gradient) and step downhill. Learning rate controls step size — too big and you overshoot, too small and it takes forever.

Types of Networks

  • Feed-forward (MLP): data flows one direction — simple, used for tabular data
  • Convolutional (CNN): uses filters that slide across data — dominant for images
  • Recurrent (RNN/LSTM): has memory of previous inputs — good for sequences, text, time series
  • Transformer: attention mechanism — the architecture behind GPT, BERT

The Universal Approximation Theorem

Here's the wild part: a feed-forward network with one hidden layer can approximate any continuous function — given enough neurons. It can learn any pattern, any relationship, any mapping from inputs to outputs.

In practice, deep networks (many layers) learn more efficiently than wide ones (many neurons per layer). Hierarchy emerges: early layers learn edges, later layers learn shapes, final layers learn concepts.

What's Actually Happening

At the end of the day, a neural network is a function with millions of parameters. Training is optimization — find the parameter values that minimize the error on your training data.

The network doesn't "understand" anything. It finds statistical patterns. When GPT writes text, it's predicting the next token based on patterns it saw in training. When AlphaGo plays Go, it's evaluating board positions based on patterns from millions of games.

No magic. Just optimization at scale.

Key Takeaways

  • Neurons are simple functions: weighted sum + activation
  • Stacking layers creates hierarchy — early layers learn simple patterns, later layers learn complex ones
  • Backpropagation is gradient descent through the network — the chain rule, computed efficiently
  • ReLU is the default activation — simple, works well, avoids vanishing gradients
  • Deep networks can approximate any function — the universal approximation theorem

Neural networks are surprisingly simple at the core. The complexity comes from scale — millions of parameters, billions of examples, massive compute. That's what makes them powerful.
