How Neural Networks Work

March 10, 2026 · Deep Dive

Neural networks are the engine behind most modern underneath AI. But the hype, they're surprisingly simple mathematical objects — layers of functions that transform inputs into outputs through lots of multiplication and addition.

The Perceptron: One Neuron

The simplest neural network is a perceptron: a single neuron that takes multiple inputs, multiplies each by a weight, sums them up, and passes the result through an activation function.

output = activation(w₁x₁ + w₂x₂ + w₃x₃ + b)

Where:

  • x = input values (features)
  • w = weights (learned parameters)
  • b = bias (shift)
  • activation = non-linear function

The weights start random. Through training, they adjust to make the network produce correct outputs.

Activation Functions

Without non-linear activation functions, stacking layers would be pointless — you'd just get another linear function. Activations introduce the "kinks" that let networks learn complex patterns.

Common activations:

  • Sigmoid: σ(x) = 1 / (1 + e⁻ˣ) — outputs 0 to 1, historically popular
  • ReLU: max(0, x) — the default. Simple, works well, avoids vanishing gradients
  • Tanh: (eˣ - e⁻ˣ) / (eˣ + e⁻ˣ) — outputs -1 to 1, zero-centered
  • GELU: used in Transformers, smoother than ReLU

Architecture Types

Feedforward (Dense)

Every neuron connects to every neuron in the next layer. Information flows one direction. Good for tabular data, simple classification.

Convolutional (CNN)

Uses filters that slide across input. Each filter learns a pattern (edges, textures, shapes). Revolutionary for image processing.

Recurrent (RNN/LSTM/GRU)

Connections loop back — has "memory." Sequences feed back into themselves. Good for time series, text, anything with temporal dependency.

Transformer

Self-attention mechanism — every position attends to every other position. Powers GPT, BERT, and most modern LLMs. The architecture that changed everything.

Backpropagation: How Networks Learn

Backpropagation is the algorithm that makes learning possible. It has two passes:

Forward pass:

input → hidden1 → hidden2 → output

Compute the prediction. Compare to actual answer. Calculate error (loss).

Backward pass:

Propagate error backwards through the network. For each weight, calculate how much that weight contributed to the error. Then adjust the weight slightly in the direction that reduces error.

weight_new = weight_old - learning_rate × gradient

The learning rate controls step size. Too big = overshoot. Too small = forever to converge.

Chain Rule in Practice

Backprop uses the chain rule from calculus. If output depends on A, which depends on B, which depends on C:

d(output)/d(C) = d(output)/d(A) × d(A)/d(B) × d(B)/d(C)

Each layer multiplies gradients as they flow back. This is why deep networks had vanishing gradients — gradients shrink exponentially through layers, making early layers hard to train.

Loss Functions

The loss measures how wrong the network is. Training = minimizing loss.

  • Mean Squared Error (MSE): regression tasks, penalizes big mistakes heavily
  • Cross-Entropy: classification, compares probability distributions
  • Hinge Loss: SVM-style, max-margin classification

Optimizers

Basic gradient descent updates all weights after seeing the full dataset (slow). Variants make it faster:

  • SGD: update after each sample (noisy but escapes local minima)
  • Momentum: adds "velocity" — builds speed in consistent directions
  • Adam: adaptive learning rates per parameter. The workhorse optimizer.

Overfitting: The Big Problem

A network can memorize training data perfectly but fail on new data. This is overfitting.

Solutions:

  • Regularization: penalize large weights
  • Dropout: randomly disable neurons during training
  • Early stopping: stop when validation loss starts rising
  • More data: the best defense

Why It Works

Here's the surprising part: we don't fully know why deep networks work so well. The lottery ticket hypothesis suggests that within a large random network, only some subnetworks are "winners" — initialized with lucky weights that can actually learn. Training prunes away the rest.

Another view: neural networks are universal function approximators. Given enough parameters, they can approximate any continuous function. Whether they find the right one depends on initialization, data, and optimization.

The Bottom Line

Neural networks are:

  • Simple mathematical functions stacked in layers
  • Trained by gradient descent (backpropagation)
  • Universal approximators — can learn any pattern given enough data and compute
  • Prone to overfitting without regularization

Everything fancy in AI — transformers, GPT, image generators — is built from these same pieces. The magic isn't in the math. It's in the scale.
