How Neural Networks Work

2026-03-09

Neural networks are the backbone of modern AI. But despite the hype, they're surprisingly simple at their core. Let's tear apart the machinery.

The Neuron: A Fancy Adder

At the heart of every neural network is the neuron (or perceptron). It's not biological — it's math:

output = activation(Σ(wi × xi) + b)

That's it. Take some inputs, multiply each by a weight, add them up, add a bias, then pass it through an activation function. The weights tell you how important each input is. The bias shifts the threshold. The activation function introduces non-linearity.

Why Non-Linearity Matters

If you skip the activation function, stacking layers just becomes a fancy way to do linear regression. You need non-linearity to model complex patterns. That's where functions like ReLU, sigmoid, and tanh come in:

  • ReLU: max(0, x) — the workhorse. Simple, fast, works well.
  • Sigmoid: 1 / (1 + e^-x) — squashes to [0, 1]. Good for probabilities.
  • Tanh: (e^x - e^-x) / (e^x + e^-x) — squashes to [-1, 1]. Zero-centered.

Architecture: Layers and Topology

Stack neurons in layers: input layer → hidden layers → output layer. Three common patterns:

  • Feedforward (MLP): Data flows one direction. Simple, predictable.
  • Convolutional (CNN): Uses filters that slide across data. Great for images.
  • Recurrent (RNN/LSTM): Has memory. Data can loop back. Good for sequences.

How Learning Works: Backpropagation

This is the magic. Backpropagation is just the chain rule from calculus, applied repeatedly:

  1. Forward pass: Run input through the network, get an output.
  2. Calculate loss: Compare output to expected result. Loss = how wrong you were.
  3. Backward pass: Propagate that error backward through the network.
  4. Update weights: Adjust each weight slightly to reduce the loss.

The "slightly" is controlled by the learning rate. Too high = overshoot. Too low = glacial convergence.

The Optimization Dance

You're trying to find the lowest point in a massive, multi-dimensional landscape (the loss surface). Simple gradient descent does this:

w = w - learning_rate × gradient

But there are smarter variants:

  • SGD: Stochastic — updates on small batches. Faster, noisier.
  • Adam: Adaptive learning rates. Usually the safest default.
  • Momentum: Builds velocity to escape local minima.

Overfitting: The Real Problem

A network can memorize training data perfectly and fail on new data. That's overfitting. Combat it with:

  • Dropout: Randomly disable neurons during training.
  • Regularization: Penalize large weights.
  • More data: The simplest solution.
  • Early stopping: Stop before you overfit.

What This Means

Neural networks aren't magic. They're function approximators — given enough neurons and the right training, they can approximate any function. The hard parts are:

  • Getting enough good data
  • Designing the right architecture
  • Tuning hyperparameters
  • Not overfitting

Everything else is just math. Beautiful, scalable, powerful math — but math nonetheless.
