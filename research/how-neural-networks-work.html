How Neural Networks Work

March 15, 2026

Neural networks are everywhere now. Every product ships with "AI-powered" this and "machine learning" that. But beneath the hype, the core concepts are surprisingly straightforward. This is my attempt to explain them clearly.

The Inspiration

The name "neural network" comes from neuroscience — the networks of neurons in your brain. But here's what most people don't realize: modern neural networks are loosely inspired by biology. They're mathematical constructs, not biological simulations. The neuron in a neural network is a simple function that takes inputs, weighs them, and produces an output.

Biological neurons fire when they reach a threshold. Artificial neurons do something similar but with continuous math.

The Building Block: The Perceptron

Let's start with the simplest possible neural network: a single neuron. In the 1950s, Frank Rosenblatt called this a perceptron.

A perceptron takes multiple inputs (x₁, x₂, x₃...), each with an associated weight (w₁, w₂, w₃...). It computes a weighted sum:

z = w₁x₁ + w₂x₂ + w₃x₃ + ... + b

Where b is a bias term — think of it as the neuron's tendency to fire regardless of input.

Then it passes this sum through an activation function to produce the output:

y = activation(z)

For simple perceptrons, the activation was a step function: fire if z > 0, don't fire otherwise. But that gave us only two possible outputs — not very useful for nuanced problems. Modern networks use smoother functions.

Activation Functions

The activation function introduces non-linearity. Without it, stacking multiple layers would be mathematically equivalent to a single layer — useless for complex problems.

Sigmoid

σ(z) = 1 / (1 + e^-z)

Outputs between 0 and 1. Historically popular, now mostly used for output layers where you need probabilities.

ReLU (Rectified Linear Unit)

ReLU(z) = max(0, z)

Zero for negative inputs, linear for positive. The workhorse of modern deep learning. Simple, fast, and works remarkably well.

Tanh

tanh(z) = (e^z - e^-z) / (e^z + e^-z)

Outputs between -1 and 1. Useful when you need values centered around zero.

Architecture: Layers

A single neuron can't do much. But stack them in layers, and magic happens.

  • Input layer: Receives your data (pixels of an image, words in a sentence, sensor readings)
  • Hidden layers: The magic happens here. Each layer transforms the output of the previous layer into something more useful
  • Output layer: Produces the final result (a classification, a prediction, a generated word)

Networks with many hidden layers are called deep — hence "deep learning."

How They Learn: Forward Propagation

When you feed data through a network, it goes layer by layer:

  1. Input enters the first hidden layer
  2. Each neuron computes its weighted sum and applies its activation function
  3. Outputs become inputs for the next layer
  4. Repeat until you reach the output layer

This is called forward propagation. It's just computation — matrix multiplications and function applications.

How They Learn: Backpropagation

The real innovation that made deep learning possible is backpropagation. Here's how it works:

  1. Make a prediction (forward pass)
  2. Compare prediction to the actual answer
  3. Calculate the loss — how wrong were we?
  4. Go backwards through the network and ask: "how much did each weight contribute to this error?"
  5. Adjust each weight slightly to reduce the error

This is where the gradient comes in. We compute the gradient of the loss with respect to each weight — essentially asking "if I increase this weight slightly, how does the loss change?" Then we adjust the weight in the direction that reduces loss.

The Chain Rule

Backpropagation uses the chain rule from calculus. If layer 3 depends on layer 2, which depends on layer 1, then the gradient at layer 1 is the product of all the gradients in between. This is why deep networks were computationally expensive before GPUs came along — you have to propagate gradients all the way back through every layer.

Learning Rate

We don't jump directly to the optimal weights. We take small steps in the right direction, controlled by the learning rate. Too high, and you overshoot the optimal solution. Too low, and training takes forever.

Optimization

Basic gradient descent has problems. The loss landscape can have local minima (valleys that aren't the lowest point), saddle points, and plateaus. Several tricks help:

Momentum

Think of a ball rolling down a hill. It builds speed and can roll through small bumps. Momentum does the same — it accumulates past gradients to help escape local minima.

Adam (Adaptive Moment Estimation)

The most popular optimizer. It adapts the learning rate for each parameter based on how much the gradient has been changing. Handles sparse gradients well and usually just works out of the box.

Learning Rate Scheduling

Start with a higher learning rate to move fast, then gradually decrease it to fine-tune as you get closer to the optimum.

Regularization: Preventing Overfitting

A network with enough parameters can memorize your training data perfectly but fail on new data. This is overfitting. Several techniques combat it:

Dropout

Randomly "turn off" some neurons during training. Forces the network to learn redundant representations and not rely on any single neuron.

L2 Regularization (Weight Decay)

Penalize large weights. Simpler models generalize better.

Data Augmentation

Transform training data (rotate images, add noise to text) to increase effective dataset size.

Early Stopping

Monitor validation loss. When it starts increasing while training loss keeps decreasing, stop. You've started overfitting.

Common Architectures

Feedforward (Dense) Networks

Every neuron connects to every neuron in the next layer. Good for tabular data, simple classification.

Convolutional Neural Networks (CNNs)

Use convolutional layers that slide filters across the input. Revolutionized image recognition. The filter detects edges, textures, shapes — hierarchical features.

Recurrent Neural Networks (RNNs)

Have connections that loop back, allowing information to persist. Good for sequences — text, time series. But suffer from vanishing gradients (hard to learn long-term dependencies).

LSTM / GRU

Improved RNNs with gating mechanisms that control what information passes through. Solve the vanishing gradient problem somewhat.

Transformers

The architecture behind GPT, BERT, and most modern LLMs. Use self-attention to weigh the importance of different parts of the input. Parallelizable, scalable, and incredibly powerful.

What Actually Happens in Practice

Here's the uncomfortable truth: we don't fully understand why deep networks work as well as they do. The theory lags behind the practice. We have empirical evidence that more data and more parameters help. We know certain architectures work better for certain tasks. But the mathematical understanding is incomplete.

Neural networks are universal function approximators — given enough neurons, they can approximate any continuous function. But that doesn't explain why gradient descent finds good solutions in the massive, high-dimensional spaces where we train them. The loss landscape of a modern language model has more dimensions than there are atoms in the universe. And yet, somehow, SGD finds minima that generalize.

That's either a miracle or a sign we have more to learn. Probably both.
