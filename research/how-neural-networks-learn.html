How Neural Networks Learn

2026-03-05

Neural networks aren't magic. They're just very elaborate function approximators with a learning procedure that's both elegant and surprisingly intuitive once you see behind the curtain.

The Core Problem

Given inputs x and known outputs y, find a function f such that f(x) ≈ y. For a neural network, f is parameterized by weights W and biases b. We want to find the right numbers in those matrices.

The problem: there's no closed-form solution. We can't solve for W directly. Instead, we iterate.

Loss: How Wrong Are We?

The first ingredient is a loss function — a single number that tells us how bad our current predictions are. Common choices:

  • Mean Squared Error (MSE): (1/n) Σ(prediction - target)² — used for regression
  • Cross-entropy: -Σ target · log(prediction) — used for classification

Think of loss as the altitude on a mountainous terrain. We want to get to the lowest point — the global minimum (or at least a local one).

Gradient Descent: Rolling Downhill

Here's the key insight: the gradient points uphill. So if we move in the opposite direction, we go downhill. This is gradient descent:

W = W - learning_rate * ∂L/∂W

The learning rate controls step size. Too big = overshoot. Too small = forever.

But computing the gradient for every data point is expensive. Stochastic Gradient Descent (SGD) estimates the gradient from a random batch — fast and works surprisingly well.

Backpropagation: The Chain Rule at Work

Now the magic. We need ∂L/∂W for every weight. The network is just a chain of functions:

output = σ(W3 · σ(W2 · σ(W1 · x + b1) + b2) + b3)

Where σ is the activation function (ReLU, sigmoid, etc.).

Backpropagation applies the chain rule from the end to the beginning:

  1. Forward pass: Compute the output and cache all intermediate values
  2. Backward pass: Compute gradients from loss back to input, using the chain rule at each layer

Each layer's gradient depends on the next layer's gradient — hence "backpropagation."

Why It Works (Mostly)

Theoretical guarantees are weak — we can't prove gradient descent finds the global minimum. But in practice:

  • Loss landscapes in high dimensions have surprisingly many flat regions
  • SGD's noise actually helps escape sharp local minima
  • Modern architectures (ResNet, transformers) have smooth loss landscapes

It's empirical alchemy, but it works.

What's Actually Happening

Training a neural network is:

  1. Forward: Compute predictions
  2. Loss: Measure error
  3. Backward: Propagate error gradients back through the network
  4. Update: Adjust weights slightly in the direction that reduces loss
  5. Repeat millions of times

That's it. Every "learning" moment in a neural network is just calculus — partial derivatives and the chain rule, applied 10,000 times a second.

Key Takeaways

  • Neural networks learn via gradient descent — minimize a loss function
  • Backpropagation is just the chain rule, computed efficiently via dynamic programming
  • The "learning" is incremental weight adjustments, not symbolic reasoning
  • No magic — just optimization in very high-dimensional space
