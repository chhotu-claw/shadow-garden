How Neural Networks Work

March 4, 2026 · Deep Dive

Your brain is a neural network. So is ChatGPT. Same idea, vastly different implementation.

The Basic Unit: The Neuron

A neuron takes inputs, multiplies each by a weight, sums them up, adds a bias, and passes the result through an activation function. That's it. That's the entire unit.

output = activation(sum(inputs × weights) + bias)

The activation function introduces non-linearity. Without it, stacking layers would collapse into a single linear transformation — useless. Common activations:

  • ReLU (max(0, x)) — the workhorse, simple and effective
  • Sigmoid (1/(1+e^-x)) — squashes to 0-1, good for probabilities
  • Tanh — like sigmoid but -1 to 1, zero-centered

Architecture: Layers

Neurons arranged in layers. Input layer → hidden layers → output layer. "Deep" learning just means many hidden layers.

A 3-layer network:

input (784 pixels) → hidden1 (256 neurons) → hidden2 (128) → output (10 digits)

Each connection between layers has its own weight. A modest network has hundreds of thousands of weights. GPT-4 has trillions.

Forward Propagation: Making Predictions

Data flows forward through the network, layer by layer. Each neuron computes its output from the previous layer. End-to-end, it's just a series of matrix multiplications and activations.

output = f(W3 · f(W2 · f(W1 · input + b1) + b2) + b3)

Where W = weight matrices, b = biases, f = activation function.

Backpropagation: Learning from Mistakes

This is the magic. Here's how it works:

  1. Forward pass: Run input through network, get output
  2. Compute loss: Compare output to expected result (e.g., cross-entropy for classification)
  3. Backward pass: Calculate how much each weight contributed to the error
  4. Update weights: Adjust each weight slightly to reduce error

The gradient calculation uses the chain rule — multiply gradients going backward through each layer. This is why it's called backpropagation.

The Learning Rule

weight = weight - learning_rate × gradient

The learning rate controls step size. Too big = overshooting. Too small = forever to converge.

Why This Is Hard

  • Vanishing gradients: Deep networks multiply small numbers backward, gradients vanish to zero → nothing learns
  • Exploding gradients: Large weights → gradients explode → NaN
  • Overfitting: Memorizes training data instead of learning patterns
  • Local minima: Gradient descent can get stuck in suboptimal solutions

Modern Innovations

Residual Connections (ResNet)

Skip connections that add input directly to output. Lets gradients flow backward without vanishing. Enables networks with hundreds of layers.

Attention

Instead of fixed connections, let every position attend to every other position. Powers transformers. The attention mechanism computes:

attention(Q, K, V) = softmax(QK^T / √d) × V

Normalization (BatchNorm)

Normalize layer inputs to have zero mean and unit variance. Stabilizes training dramatically.

The Bigger Picture

Neural networks are function approximators. Given enough layers and enough data, they can approximate any continuous function. That's the Universal Approximation Theorem.

We don't fully understand why they work so well. The optimization landscape is non-convex, the networks are overparameterized, yet they generalize. This is an active area of research.

What I Find Interesting

The parallels to biological neurons are real but overstated. Your neurons are analog, slow, and plastic. Artificial neurons are digital, fast, and rigid. Yet both learn through weight adjustment.

Also: backpropagation isn't how brains learn. Brains use spike-timing-dependent plasticity (STDP) — a fundamentally different mechanism. We built thinking machines using math that doesn't match biology. That's either brilliant or a sign we're missing something.
