How Neural Networks Work

March 3, 2026

Your brain rewires itself based on what you think and do. Neurons that fire together, wire together. That biological intuition — cells that change their connections based on experience — is exactly what artificial neural networks do. Except in silicon.

The Basic Unit: The Neuron

A neuron takes inputs, multiplies each by a weight, sums them up, adds a bias, and runs the result through an activation function. That's it. A single neuron is just:

output = activation(sum(inputs × weights) + bias)

The activation function introduces non-linearity — without it, stacking layers would collapse into a single linear transformation. Common activations:

  • Sigmoid: squashes to (0,1) — good for probabilities
  • ReLU: max(0, x) — the workhorse of modern networks
  • Tanh: squashes to (-1,1) — zero-centered

Layers: Stacking the Units

Neurons arranged in layers. Input layer receives data (pixels, audio, text). Output layer produces predictions. Layers in between are hidden — nobody sees what they learn.

A fully connected layer: every neuron connects to every neuron in the next layer. Each connection has its own weight. A modern vision network might have millions of these weights. A large language model has billions.

Forward Pass: Computing an Output

Data flows forward: input → hidden layers → output. Each layer transforms the representation. Early layers learn edges and textures. Deeper layers learn concepts like "face" or "word." This emergent hierarchy is what makes deep learning powerful — no one programs "face detector." It emerges from learning.

# Simplified forward pass
for layer in network:
    output = activation(layer.weights @ input + layer.bias)
    input = output  # pass to next layer

Loss: Measuring the Error

How wrong was the network? Loss measures the difference between prediction and truth.

  • Cross-entropy: for classification — "how surprised was the model?"
  • Mean squared error: for regression — "how far off was the prediction?"

The loss function defines what "learning" means. Change the loss, change what the network learns to do.

Backpropagation: The Magic

Backpropagation is just the chain rule from calculus, applied repeatedly. It computes how each weight contributed to the error, then adjusts the weight in the direction that reduces loss.

loss.backward()  # compute gradients
optimizer.step()  # update weights
optimizer.zero_grad()  # reset for next iteration

That's it. Four lines of code that, when repeated millions of times on millions of examples, produce systems that can translate languages, generate art, and play chess better than humans.

Gradient Descent: The Update Rule

Gradients point uphill. We go downhill. Learning rate controls step size:

  • Too small: learning is painfully slow
  • Too large: overshoot, oscillate, diverge

Practical training uses tricks: learning rate schedules, Adam optimizer (adaptive per-parameter rates), momentum (continuing in the same direction).

Overfitting: The Central Problem

A network that memorizes training data learns nothing useful. Regularization techniques fight this:

  • Dropout: randomly disable neurons during training
  • Weight decay: penalize large weights
  • Data augmentation: transform training examples

The gap between training loss and validation loss tells you everything. If training keeps dropping but validation stalls — you've overfit.

Architectures: What Works

  • CNNs: convolutional networks — spatially aware, great for images
  • RNNs/LSTMs/GRUs: sequential data, memory of previous tokens
  • Transformers: attention mechanism — current state of the art for text, images, audio

The Core Insight

Neural networks are function approximators. Given enough data and the right architecture, they can approximate any continuous function. That's the universal approximation theorem — mathematically proven.

What emerged from this simple idea:

  • GPT, Claude, Gemini — transformers predicting next tokens
  • Stable Diffusion — predicting pixels from noise
  • AlphaGo — policy and value networks for game tree search
  • Voice synthesis — waveform generation from text

All the same math. Forward pass, loss, backprop, update. Repeat 10^23 times across 10^11 parameters. Emergence.

What Nobody Understands

We know that it works. We don't fully understand why it works so well. The optimization landscape is non-convex — theoretically intractable. Yet in practice, SGD finds good minima. Why? Theories abound: implicit regularization, ensembling through weight decay, the lottery ticket hypothesis. But definitive answers? Still missing.

The gap between "we can build it" and "we understand why it works" is where the interesting research lives.
